PROYECTO FINAL DE FUNDAMENTOS DE SOFTWARE

1. Algoritmo

Un algoritmo es una secuencia finita de pasos lógicos que resuelven un problema o ejecutan una tarea específica. Es la base de la programación, ya que convierte una necesidad en instrucciones precisas que la computadora puede seguir.


2. Lenguaje de programación

Un lenguaje de programación es un medio estructurado que permite a los desarrolladores comunicarse con los sistemas informáticos mediante reglas sintácticas y semánticas. Gracias a ellos se crean aplicaciones, páginas web y programas.


3. Variable

Una variable es un espacio de memoria identificado con un nombre, cuyo contenido puede cambiar a lo largo de la ejecución de un programa. Es esencial para manejar datos dinámicos.


4. Constante

Una constante es un valor almacenado en la memoria que no puede modificarse mientras el programa se ejecuta. Se utiliza para asegurar que ciertos datos permanezcan fijos.


5. Estructura de datos

Las estructuras de datos organizan la información en formas específicas para optimizar su almacenamiento y procesamiento. Ejemplos son listas, pilas, colas y árboles.


6. Base de datos

Una base de datos es un sistema digital que organiza información de forma estructurada, permitiendo la consulta, modificación y eliminación de datos de manera rápida y segura.


7. Compilador

Un compilador traduce el código fuente de un lenguaje de alto nivel al lenguaje de máquina. Además de traducir, verifica errores de sintaxis y optimiza la ejecución.


8. Intérprete

Un intérprete ejecuta directamente el código fuente línea a línea, lo que facilita la detección inmediata de errores y el trabajo interactivo.


9. Framework

Un framework es un entorno de trabajo compuesto por librerías, herramientas y normas que brindan al programador una base sólida para desarrollar aplicaciones más rápido.


10. Paradigma de programación

Un paradigma de programación es un enfoque que define cómo estructurar y organizar el código. Ejemplos son la programación orientada a objetos, funcional y estructurada.


11. Repositorio

Un repositorio es un espacio digital en el que se almacena el código fuente de un proyecto, junto con su historial de cambios y documentación.


12. Control de versiones

El control de versiones permite gestionar el historial de modificaciones de un proyecto. Gracias a él es posible restaurar versiones anteriores y trabajar en equipo de manera organizada.


13. Git

Git es un sistema distribuido de control de versiones que registra cambios en los archivos y permite colaborar de manera simultánea en un mismo proyecto.


14. GitHub

GitHub es una plataforma en línea que aloja repositorios y agrega funciones colaborativas como revisión de código y gestión de proyectos.


15. IDE

Un entorno de desarrollo integrado (IDE) reúne en una sola aplicación herramientas como editor de texto, depurador, compilador y gestor de proyectos.


16. Depuración

La depuración es el proceso sistemático de encontrar, analizar y corregir errores en un programa para garantizar su correcto funcionamiento.


17. Algoritmo de búsqueda

Un algoritmo de búsqueda es un procedimiento diseñado para localizar un elemento dentro de una colección de datos. Puede ser lineal o binario, entre otros.


18. Algoritmo de ordenamiento

Estos algoritmos reorganizan los datos en un orden específico, como ascendente o descendente. Algunos conocidos son el burbuja, inserción y quicksort.


19. Pila

Una pila es una estructura de datos que sigue el principio de último en entrar, primero en salir. Es útil en funciones recursivas y sistemas de deshacer.


20. Cola

Una cola es una estructura que opera bajo el principio primero en entrar, primero en salir. Es común en la gestión de procesos y servicios en espera.


21. Árbol binario

Un árbol binario organiza datos en una estructura jerárquica donde cada nodo tiene como máximo dos hijos. Facilita búsquedas rápidas y ordenadas.


22. Grafo

Un grafo es un conjunto de nodos y conexiones que representan relaciones entre elementos. Se utiliza en redes sociales, rutas y análisis de conexiones.


23. Nodo

Un nodo es la unidad básica de estructuras como listas, árboles o grafos, y contiene información junto con referencias hacia otros nodos.


24. Bucle

Un bucle es una estructura de control que repite un bloque de código mientras se cumpla una condición determinada.


25. Función

Una función es un bloque de código independiente que recibe parámetros, ejecuta instrucciones y devuelve un resultado. Promueve la reutilización del código.


26. Procedimiento

Un procedimiento es similar a una función, pero no devuelve un valor. Se emplea para agrupar instrucciones que cumplen una tarea específica.


27. Clases

Una clase es la plantilla en la que se definen los atributos y métodos de los objetos en la programación orientada a objetos.


28. Objeto

Un objeto es una instancia de una clase que combina datos y comportamientos, representando entidades del mundo real dentro del software.


29. Herencia

La herencia permite que una clase derive de otra, compartiendo sus atributos y métodos. Facilita la reutilización y extensión del código.


30. Polimorfismo

El polimorfismo permite que una misma función o método se comporte de distintas formas según el contexto o los objetos que lo usen.


31. Encapsulamiento

El encapsulamiento protege los datos de un objeto, permitiendo acceder a ellos únicamente a través de métodos definidos.


32. Abstracción

La abstracción consiste en ocultar los detalles de implementación de un objeto y mostrar solo la información esencial para su uso.


33. API

Una API es un conjunto de reglas que permite que distintos programas se comuniquen entre sí, compartiendo funciones y datos de manera segura.


34. Microservicios

Los microservicios son un enfoque de arquitectura de software en el que las aplicaciones se dividen en pequeños servicios independientes que se comunican entre sí.


35. Arquitectura cliente-servidor

Este modelo distribuye responsabilidades: el cliente solicita información y el servidor la procesa y responde.


36. Arquitectura en capas

Es un diseño de software en el que el sistema se organiza en capas, como presentación, lógica de negocio y datos, para mejorar la separación de responsabilidades.


37. Cloud Computing

La computación en la nube es un modelo que ofrece recursos informáticos como almacenamiento y procesamiento a través de internet bajo demanda.


38. Virtualización

La virtualización permite crear versiones simuladas de recursos físicos, como servidores o redes, para optimizar su uso y flexibilidad.


39. Contenedor

Un contenedor es un entorno ligero que encapsula una aplicación y sus dependencias, garantizando que se ejecute de forma consistente en distintos entornos.


40. Docker

Docker es una plataforma que automatiza el despliegue de aplicaciones en contenedores, mejorando la portabilidad y escalabilidad de los sistemas.


41. Kubernetes

Kubernetes es una plataforma de orquestación de contenedores que automatiza la implementación, escalado y gestión de aplicaciones distribuidas, facilitando la administración de sistemas complejos.


42. Sistema operativo

Un sistema operativo es el software principal que administra los recursos físicos y virtuales de un computador, permitiendo la interacción entre hardware, programas y usuarios.


43. Kernel

El kernel es el núcleo de un sistema operativo que controla la comunicación entre el hardware y el software, gestionando procesos, memoria y dispositivos.


44. Interfaz gráfica de usuario

Una interfaz gráfica de usuario (GUI) permite que las personas interactúen con un sistema mediante iconos, ventanas y menús en lugar de comandos de texto.


45. Interfaz de línea de comandos

La CLI es una forma de comunicación con el sistema mediante comandos escritos, ofreciendo un control más preciso para usuarios avanzados.


46. Protocolo

Un protocolo es un conjunto de normas que regulan cómo se transmiten y reciben datos en una red, asegurando la correcta comunicación entre dispositivos.


47. HTTP

El protocolo HTTP permite la transferencia de información entre servidores web y navegadores, siendo la base de la comunicación en internet.


48. HTTPS

HTTPS es la versión segura de HTTP que cifra la información transmitida, garantizando la privacidad e integridad de los datos en línea.


49. IP

Una dirección IP identifica de manera única a cada dispositivo conectado a una red, permitiendo el envío y recepción de información.


50. DNS

El DNS traduce nombres de dominio legibles por humanos en direcciones IP, haciendo posible navegar en internet usando nombres fáciles de recordar.


51. Servidor web

Un servidor web es un software que entrega páginas y recursos solicitados por los usuarios a través de un navegador.


52. Navegador web

El navegador es la aplicación que permite acceder y visualizar sitios en internet, interpretando lenguajes como HTML, CSS y JavaScript.


53. HTML

El HTML es el lenguaje de marcado que estructura el contenido de las páginas web, organizando textos, imágenes, enlaces y otros elementos.


54. CSS

Las hojas de estilo en cascada (CSS) permiten definir la apariencia visual de un sitio web, controlando colores, tamaños, fuentes y diseño.


55. JavaScript

JavaScript es un lenguaje de programación que añade dinamismo a los sitios web, permitiendo interactividad, validaciones y animaciones.


56. Frontend

El frontend es la parte visible de una aplicación o sitio web, enfocada en la experiencia del usuario mediante diseño e interacción.


57. Backend

El backend es la parte de un sistema que procesa datos, ejecuta lógica de negocio y gestiona la comunicación con las bases de datos.


58. Fullstack

Un desarrollador fullstack es aquel que domina tanto las tecnologías del frontend como las del backend, pudiendo crear aplicaciones completas.


59. Arquitectura monolítica

La arquitectura monolítica es un modelo en el que toda la aplicación se desarrolla como un único bloque de código, lo que puede dificultar su escalado.


60. Arquitectura orientada a servicios

Este enfoque divide un sistema en servicios independientes que se comunican entre sí mediante protocolos estándar.


61. Inteligencia artificial

La inteligencia artificial es la disciplina que busca crear sistemas capaces de simular el razonamiento y aprendizaje humano mediante algoritmos avanzados.


62. Aprendizaje automático

El machine learning es una rama de la inteligencia artificial que permite a los sistemas mejorar su rendimiento a partir de datos sin programación explícita.


63. Red neuronal artificial

Una red neuronal artificial imita el funcionamiento del cerebro humano a través de capas de nodos interconectados que procesan información.


64. Big Data

El Big Data se refiere al tratamiento de volúmenes masivos de datos que no pueden ser procesados con métodos tradicionales por su tamaño y complejidad.


65. Analítica de datos

La analítica de datos es el proceso de examinar conjuntos de información para identificar patrones, tendencias y tomar decisiones fundamentadas.


66. Ciencia de datos

La ciencia de datos combina estadística, matemáticas y programación para extraer conocimiento útil de grandes volúmenes de datos.


67. Minería de datos

La minería de datos consiste en descubrir información oculta en grandes conjuntos de datos mediante algoritmos y modelos predictivos.


68. Business Intelligence

El Business Intelligence agrupa tecnologías y prácticas que convierten datos en información valiosa para la gestión empresarial.


69. Criptografía

La criptografía es el estudio de técnicas para proteger la información mediante el uso de códigos y cifrados.


70. Cifrado simétrico

Este tipo de cifrado utiliza una misma clave para encriptar y desencriptar datos, lo que requiere mantenerla segura entre las partes.


71. Cifrado asimétrico

El cifrado asimétrico usa un par de claves (pública y privada) para proteger la información, facilitando la comunicación segura en internet.


72. Firma digital

Una firma digital es un mecanismo criptográfico que garantiza la autenticidad de un documento electrónico y la identidad de su emisor.


73. Blockchain

El blockchain es una tecnología de registro distribuido que almacena transacciones en bloques encadenados, imposibilitando su alteración.


74. Contrato inteligente

Un contrato inteligente es un programa en blockchain que se ejecuta automáticamente al cumplirse ciertas condiciones predefinidas.


75. Ciberseguridad

La ciberseguridad abarca medidas y prácticas destinadas a proteger sistemas y redes de ataques y accesos no autorizados.


76. Malware

El malware es un tipo de software malicioso diseñado para dañar, robar o interrumpir el funcionamiento de un sistema.


77. Phishing

El phishing es una técnica de fraude en la que se engaña a los usuarios para obtener información sensible mediante correos o páginas falsas.


78. Ransomware

El ransomware es un tipo de malware que cifra los archivos de un sistema y exige un pago para recuperarlos.


79. Firewall

Un firewall es un sistema de seguridad que controla el tráfico de red, bloqueando accesos no autorizados y permitiendo conexiones seguras.


80. Antivirus

El antivirus es un software que detecta, bloquea y elimina programas maliciosos para proteger el sistema del usuario.


81. Pruebas de software

Las pruebas de software son procesos de verificación diseñados para garantizar que un programa cumple con los requisitos y funciona sin errores.


82. Prueba unitaria

Las pruebas unitarias evalúan pequeñas partes del código, como funciones o métodos, para comprobar que funcionan de manera aislada.


83. Prueba de integración

Este tipo de prueba verifica que los distintos módulos de un sistema funcionen correctamente al interactuar entre sí.


84. Prueba de sistema

Las pruebas de sistema validan que el software completo cumpla con los requisitos funcionales y no funcionales definidos.


85. Prueba de aceptación

Son pruebas realizadas por los usuarios finales para validar que el software satisface sus necesidades reales.


86. Desarrollo ágil

El desarrollo ágil es una metodología que prioriza la colaboración, la adaptabilidad y la entrega rápida de software funcional.


87. Scrum

Scrum es un marco de trabajo ágil que organiza el desarrollo en iteraciones cortas llamadas sprints, con roles y reuniones definidos.


88. Kanban

Kanban es una metodología visual de gestión de tareas que usa tableros con tarjetas para organizar el flujo de trabajo.


89. Sprint

Un sprint es un ciclo corto de trabajo en Scrum, generalmente de dos a cuatro semanas, en el que se entrega un incremento del producto.


90. Product Owner

El Product Owner es el responsable de definir las prioridades del producto en Scrum, asegurando que el desarrollo aporte valor al cliente.


91. Scrum Master

El Scrum Master guía al equipo en la aplicación de Scrum, eliminando obstáculos y asegurando la correcta adopción del marco de trabajo.


92. Historias de usuario

Las historias de usuario describen de manera breve las funcionalidades que el cliente necesita, expresadas en lenguaje natural.


93. Prototipo

Un prototipo es una representación temprana y simplificada de un producto que permite validar ideas y recopilar retroalimentación.


94. Diseño de interfaz

El diseño de interfaz se enfoca en crear pantallas y elementos visuales que permitan una interacción sencilla y atractiva con el usuario.


95. Experiencia de usuario

La experiencia de usuario (UX) mide la percepción global del usuario al interactuar con un sistema, considerando facilidad, utilidad y satisfacción.


96. Accesibilidad digital

La accesibilidad digital busca que aplicaciones y sitios web puedan ser usados por personas con diferentes capacidades o limitaciones.


97. Ingeniería de software

La ingeniería de software es la disciplina que aplica principios de ingeniería al diseño, desarrollo y mantenimiento de sistemas informáticos.


98. Ciclo de vida del software

El ciclo de vida del software describe las fases por las que pasa un sistema: análisis, diseño, desarrollo, pruebas, implementación y mantenimiento.


99. Mantenimiento de software

El mantenimiento consiste en actualizar, corregir y mejorar un sistema ya desarrollado para prolongar su vida útil y adaptarlo a nuevas necesidades.


100. Documentación técnica

La documentación técnica es el conjunto de manuales, diagramas y descripciones que explican cómo funciona y cómo debe usarse un software.


101. Modelo en cascada

El modelo en cascada es una metodología de desarrollo de software secuencial, donde cada fase debe completarse por completo antes de avanzar a la siguiente, lo que facilita la planificación pero limita la flexibilidad.


102. Modelo incremental

El modelo incremental divide el desarrollo en pequeñas entregas sucesivas que van construyendo el sistema completo, lo que permite recibir retroalimentación temprana y reducir riesgos.


103. Modelo espiral

El modelo espiral combina elementos del desarrollo en cascada e iterativo, incorporando análisis de riesgos en cada ciclo para mejorar la calidad del producto.


104. DevOps

DevOps es una cultura y conjunto de prácticas que integra desarrollo y operaciones, fomentando la automatización, la colaboración y la entrega continua de software.


105. Integración continua

La integración continua consiste en fusionar los cambios de código de varios desarrolladores en un repositorio compartido de manera frecuente para detectar errores de forma temprana.


106. Entrega continua

La entrega continua extiende la integración continua asegurando que el software siempre esté en un estado desplegable, permitiendo liberaciones rápidas y confiables.


107. Despliegue continuo

El despliegue continuo automatiza el lanzamiento de aplicaciones en entornos de producción sin intervención manual, reduciendo tiempos de entrega.


108. Refactorización

La refactorización es el proceso de reestructurar el código de un programa sin alterar su funcionalidad, con el fin de mejorar su legibilidad, eficiencia y mantenibilidad.


109. Testing automatizado

El testing automatizado emplea herramientas que ejecutan pruebas de software de manera automática, reduciendo errores humanos y acelerando la validación.


110. Mocking

El mocking es una técnica de pruebas que simula el comportamiento de componentes externos para probar una parte específica del sistema en aislamiento.


111. Caso de uso

Un caso de uso describe cómo un usuario interactúa con un sistema para lograr un objetivo, siendo una herramienta clave en el análisis de requisitos.


112. Diagrama de clases

Un diagrama de clases es una representación visual de la estructura de un sistema orientado a objetos, mostrando atributos, métodos y relaciones.


113. Diagrama de secuencia

Este diagrama ilustra cómo los objetos de un sistema se comunican a lo largo del tiempo, detallando el orden de los mensajes.


114. Diagrama de flujo

Un diagrama de flujo utiliza símbolos gráficos para representar el recorrido lógico de un proceso o algoritmo.


115. UML

El UML es un lenguaje de modelado estándar que permite visualizar, especificar y documentar los componentes de un sistema de software.


116. Caso de prueba

Un caso de prueba es una condición definida que permite verificar si un sistema cumple correctamente con un requisito específico.


117. Requisito funcional

Los requisitos funcionales describen las acciones que un sistema debe realizar, como cálculos, validaciones o generación de reportes.


118. Requisito no funcional

Los requisitos no funcionales definen características de calidad como seguridad, rendimiento, usabilidad y disponibilidad del sistema.


119. Análisis de requisitos

El análisis de requisitos es la fase en la que se recopila, estudia y define lo que el cliente necesita que el software realice.


120. Especificación de software

La especificación de software documenta de manera formal y detallada las funciones y restricciones que tendrá un sistema.


121. Arquitectura de software

La arquitectura de software es el diseño estructural que define los componentes principales de un sistema y sus interacciones.


122. Middleware

El middleware es un software intermedio que facilita la comunicación y gestión de datos entre aplicaciones o sistemas diferentes.


123. API REST

Una API REST es una interfaz basada en el protocolo HTTP que permite la comunicación entre aplicaciones de forma sencilla y escalable.


124. API SOAP

SOAP es un protocolo basado en XML que estandariza el intercambio de mensajes entre sistemas distribuidos.


125. Web Service

Un web service es un servicio disponible en internet que permite el intercambio de datos entre aplicaciones a través de protocolos estándar.


126. JSON

JSON es un formato ligero de intercambio de datos basado en texto que organiza la información en pares clave-valor, siendo fácil de leer y procesar.


127. XML

XML es un lenguaje de marcado que organiza datos mediante etiquetas, permitiendo compartir información estructurada entre sistemas.


128. YAML

YAML es un formato de serialización de datos enfocado en la legibilidad humana, ampliamente utilizado en configuraciones de sistemas.


129. Token

Un token es un elemento de seguridad que representa credenciales de acceso temporales para autenticar usuarios o aplicaciones.


130. OAuth

OAuth es un protocolo de autorización que permite a las aplicaciones acceder a recursos en nombre de un usuario sin compartir su contraseña.


131. JWT

Los JSON Web Tokens son credenciales codificadas que permiten la transmisión segura de información entre dos partes.


132. Sesión

Una sesión es el estado temporal que mantiene la interacción entre un usuario y un sistema durante un periodo determinado.


133. Cookie

Una cookie es un pequeño archivo que almacena datos en el navegador del usuario para recordar preferencias o sesiones activas.


134. Cache

La cache almacena temporalmente datos para acelerar el acceso posterior y mejorar el rendimiento de un sistema.


135. Balanceador de carga

Un balanceador distribuye el tráfico entrante entre varios servidores para optimizar recursos y asegurar la disponibilidad.


136. Alta disponibilidad

La alta disponibilidad es la capacidad de un sistema de continuar operativo incluso ante fallos, mediante redundancia y tolerancia a errores.


137. Escalabilidad

La escalabilidad es la capacidad de un sistema de aumentar su rendimiento al añadir más recursos, ya sean hardware o software.


138. Tolerancia a fallos

La tolerancia a fallos es la propiedad de un sistema de seguir funcionando, aunque algunos de sus componentes fallen.


139. Clúster

Un clúster es un conjunto de servidores conectados que funcionan como una sola unidad para mejorar rendimiento y disponibilidad.


140. Servidor de aplicaciones

Un servidor de aplicaciones es un software que provee servicios y funcionalidades para ejecutar programas de manera centralizada.


141. Virtual Machine

Una máquina virtual es una emulación de un sistema informático que se ejecuta sobre hardware físico compartido.


142. Hipervisor

El hipervisor es el software que permite crear y administrar máquinas virtuales sobre un mismo servidor físico.


143. Cloud pública

La nube pública ofrece recursos como almacenamiento y cómputo a través de proveedores externos accesibles por internet.


144. Cloud privada

La nube privada es una infraestructura de servicios en la nube dedicada a una sola organización, brindando mayor control y seguridad.


145. Cloud híbrida

La nube híbrida combina recursos de nube pública y privada para aprovechar la flexibilidad y la seguridad de ambos modelos.


146. Edge Computing

El Edge Computing procesa los datos cerca del lugar donde se generan, reduciendo la latencia y mejorando la eficiencia.


147. IoT

El Internet de las cosas conecta objetos físicos a internet para recopilar y compartir datos en tiempo real.


148. Dispositivo embebido

Un dispositivo embebido es un sistema informático especializado diseñado para realizar funciones específicas dentro de un producto.


149. Microcontrolador

Un microcontrolador es un circuito integrado que incluye procesador, memoria y periféricos, utilizado en dispositivos electrónicos.


150. FPGA

Una FPGA es un dispositivo programable que permite implementar circuitos digitales de manera flexible y personalizada.


151. Sensor

Un sensor es un componente electrónico que capta información del entorno, como temperatura, luz o movimiento, y la convierte en datos digitales.


152. Actuador

Un actuador es un dispositivo que transforma señales eléctricas en acciones físicas, como mover un motor o encender una luz.


153. API Gateway

Un API Gateway es un servidor que actúa como punto de entrada para gestionar solicitudes a múltiples servicios de una aplicación.


154. CDN

Una red de distribución de contenidos almacena copias de archivos en diferentes ubicaciones geográficas para mejorar la velocidad de acceso.


155. Latencia

La latencia es el tiempo que tarda un paquete de datos en viajar desde el origen hasta el destino en una red.


156. Ancho de banda

El ancho de banda es la capacidad máxima de transmisión de datos de una red durante un tiempo determinado.


157. Paquete de datos

Un paquete es una unidad de información dividida en bloques que viaja a través de redes para optimizar el envío.


158. Protocolo TCP/IP

El protocolo TCP/IP es la base de internet, encargado de dividir, transmitir y reensamblar los datos enviados entre computadoras.


159. Modelo OSI

El modelo OSI es una referencia que divide la comunicación en siete capas, cada una con funciones específicas para estandarizar redes.


160. Capa de aplicación

La capa de aplicación en el modelo OSI permite a los usuarios acceder a los servicios de red, como correo o navegación web.


161. Capa de presentación

Esta capa traduce los datos en un formato entendible para la aplicación, gestionando codificación y cifrado.


162. Capa de sesión

La capa de sesión establece, mantiene y finaliza la comunicación entre aplicaciones en una red.


163. Capa de transporte

La capa de transporte asegura que los datos lleguen de manera correcta y ordenada entre aplicaciones.


164. Capa de red

La capa de red se encarga de dirigir los datos desde el origen hasta el destino a través de múltiples rutas.


165. Capa de enlace

La capa de enlace gestiona la transferencia de datos entre nodos de una red local, detectando errores en la transmisión.


166. Capa física

La capa física transmite los bits en medios físicos como cables o señales inalámbricas.


167. Wireless

La tecnología inalámbrica permite la transmisión de datos sin cables, utilizando ondas de radio o infrarrojos.


168. WiFi

WiFi es una tecnología inalámbrica que conecta dispositivos a internet mediante redes de corto alcance.


169. Bluetooth

1000. array

Estructura de datos que almacena múltiples valores en una sola variable. Cada valor puede accederse mediante un índice numérico.


1001. object

Tipo de dato que almacena información en pares clave–valor. Permite representar entidades complejas como usuarios, productos o configuraciones.


1002. document.getElementById()

Método que permite acceder a un elemento del DOM utilizando su identificador único. Es ampliamente usado para modificar contenido dinámicamente.


1003. addEventListener()

Método que permite escuchar eventos del usuario, como clics o movimientos del ratón, y ejecutar funciones específicas cuando ocurren.


1004. querySelector()

Selecciona el primer elemento del DOM que coincide con un selector CSS. Es una forma moderna y flexible de acceder a elementos en una página.


1005. console.log()

Instrucción utilizada para imprimir mensajes o valores en la consola del navegador. Es una herramienta esencial para depurar y verificar el comportamiento del código.
